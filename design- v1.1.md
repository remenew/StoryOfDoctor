## 一、项目定位

### 核心陈述

一款以江湖医生成长为叙事框架的卡牌 roguelike 游戏。玩家从无名郎中开始，在江湖中行医、探索，逐步成为闻名天下的神医。

**中医的角色**：主题载体，不是内容本身。
- 东亚玩家对中医有广泛的基础认知 → 降低入门教育成本
- 君臣佐使配伍体系提供天然的卡牌组合设计空间 → 机制深度
- "知道但不熟悉"的神秘感 → 探索乐趣

这不是一款中医知识游戏。它是一个江湖故事，中医是机制语言。

### 目标玩家

**交集人群**：卡牌 roguelike 深度玩家（杀戮尖塔 / 小丑牌受众）× 江湖/武侠文化认同者。

不是中医爱好者。是在杀戮尖塔上死了 100 次、还在找新鲜感的那个人。

### 团队目标

小团队创始作品，磨合团队，做好它。

## 二、世界观与背景设定

### 时代背景

**光绪二十四年（1898年），清廷腐朽，乱世将至**

甲午战败的创伤未愈，戊戌变法刚刚失败，慈禧重新垂帘听政。列强环伺，租界扩张，传统与现代、东方与西方在这个时代激烈碰撞。

这是一个礼崩乐坏的时代：
- **朝廷**：守旧派掌权，改革派被镇压，皇权摇摇欲坠
- **官府**：贪腐横行，苛捐杂税，民不聊生
- **民间**：白莲教、义和团等民间组织兴起，民心迷乱
- **洋人**：租界林立，传教士活跃，西医开始传入

### 游戏世界

玩家扮演一位游方郎中，在这个乱世中行医求生。你的足迹将遍布：

| 区域 | 特点 | 势力分布 |
|------|------|---------|
| **朝廷** | 权力中心，危险与机遇并存 | 守旧派、改革派、外国使节 |
| **官府** | 明规则与潜规则交织 | 清官、贪官、胥吏 |
| **城镇** | 商业繁华，三教九流汇聚 | 商会、帮派、洋人 |
| **村庄** | 传统中医根基，淳朴民风 | 乡绅、农户、游方郎中 |
| **山寨** | 绿林好汉，亦正亦邪 | 义匪、恶霸、流亡者 |
| **租界** | 洋人中立区，现代医学传入 | 外国医生、传教士、买办 |

### 核心冲突

**生存与道义的抉择**

在这个乱世，你将不断面临选择：
- 是救治贫苦的百姓，还是为达官贵人服务以获取丰厚报酬？
- 是坚守医者的道德底线，还是为了生存与黑道合作？
- 是传承传统中医，还是学习西洋医术？
- 是独善其身，还是参与到时代的大变局中？

你的每一个选择都会影响四大核心属性，最终决定你成为怎样的医者，以及你将迎来怎样的结局。

### 叙事主题

**"医人易，医国难"**

作为医者，你能治愈个体的疾病；但作为乱世的见证者，你是否能治愈这个时代的创伤？

游戏通过多个结局展现不同的选择结果：
- **医道宗师**：悬壶济世，成为一代名医
- **御医院判**：进入朝廷，权力与责任的平衡
- **江湖游医**：独善其身，逍遥自在
- **邪医魔头**：为达目的不择手段
- **变革先驱**：融合中西，开创医学新篇

## 三、核心规则系统

### 3.1 生存机制 — 四大核心属性

玩家角色拥有四大核心属性，这些属性相互关联，共同决定玩家在游戏世界中的生存状态和发展方向。

#### 属性定义

| 属性 | 范围 | 说明 | 主要影响 |
|------|------|------|---------|
| **声望值** | 0-2000 | 江湖名声与社会地位 | 解锁任务等级、NPC态度、结局选择 |
| **道德值** | -100~+100 | 行为准则与立场倾向 | 正派/邪派NPC态度、部分技能解锁 |
| **健康值** | 0-100 | 身体状况 | 治疗效果、疾病debuff触发 |
| **精力值** | 0-20 | 每日行动力 | 探索次数、治疗次数限制 |

#### 属性变化机制

**声望值获取途径**：
| 行为 | 声望变化 | 备注 |
|------|---------|------|
| 成功治愈普通病人 | +5~15 | 根据病症难度 |
| 成功治愈精英/Boss | +20~50 | 重大声望来源 |
| 完成主线任务 | +30~100 | 剧情推进奖励 |
| 救助贫民（免费治疗） | +2~5 | 道德行为附加 |
| 医治黑道人物 | +5（黑道声望） | 正道声望-5 |
| 治疗失败 | -10~-30 | 重大失败惩罚 |
| 见死不救 | -20 | 道德值同步下降 |

**道德值变化机制**：
| 行为 | 道德变化 | 影响 |
|------|---------|------|
| 救助贫民 | +5~10 | 提升正派NPC好感 |
| 拒绝黑道委托 | +10 | 坚守医者底线 |
| 使用光明正大手段 | +3~5 | 常规行医 |
| 使用黑暗技能/手段 | -10~-20 | 邪派技能、毒药等 |
| 为金钱医治恶人 | -5~-15 | 根据恶行程度 |
| 见死不救 | -15 | 严重道德污点 |
| 主动伤害病人 | -30 | 邪医路线 |

**健康值维持机制**：

健康值每日自动下降，必须通过消耗金钱维持：
| 健康状态 | 健康值范围 | 每日消耗 | 效果 |
|---------|-----------|---------|------|
| 健康 | 70-100 | 10文 | 正常治疗 |
| 疲惫 | 30-69 | 5文 | 治疗效果-20% |
| 重病 | 1-29 | 无法恢复 | 触发疾病debuff，无法出诊 |
| 濒死 | 0 | — | 游戏结束（除非有复活技能）|

**健康值恢复方式**：
- 正常休息：每日+10（需消耗金钱）
- 服用补品：+20~50（消耗药材/金钱）
- 特殊事件：+10~30（奇遇、NPC帮助等）

**精力值系统**：

- **每日重置**：凌晨自动恢复至上限（基础10点）
- **上限提升**：通过技能、道具、升级可增加上限
- **消耗场景**：
  | 行为 | 精力消耗 |
  |------|---------|
  | 普通探索 | 1点 |
  | 困难区域探索 | 2点 |
  | 危险区域探索 | 3点 |
  | 普通治疗 | 0点 |
  | 精英/Boss治疗 | 1点 |
  | 采集草药 | 1点 |

#### 属性过低惩罚

**声望值过低（<50）**：
- 无法接取中级以上任务
- 部分NPC拒绝交易
- 只能治疗普通病人

**道德值过低（<-50）**：
- 正派NPC敌视，拒绝服务
- 邪派NPC主动接触，提供黑暗任务
- 解锁邪医技能树
- 可能触发正道追杀事件

**健康值过低（<30）**：
- 强制触发疾病debuff
- 无法出诊治疗
- 必须休息恢复
- 连续3日健康值<10触发死亡结局

**精力值耗尽**：
- 无法进行探索和采集
- 只能进行基础治疗
- 必须结束当日（进入下一天）

### 3.2 卡牌机制 — 三大卡牌类型

游戏卡牌分为三种类型，每种类型有不同的用途和获取方式。

#### 卡牌类型定义

| 类型 | 图标 | 用途 | 获取方式 | 存储位置 |
|------|------|------|---------|---------|
| **草药卡** | 🌿 | 治疗基础，组成药方 | 采集、购买、任务奖励 | 牌库（战斗中抽取）|
| **秘方卡** | 📜 | 特殊方剂，强力效果 | 完成治疗、探索发现、任务奖励 | 收藏册（永久持有）|
| **巫术卡** | 🔮 | 祝由术法，特殊效果 | 特定NPC、黑市、任务奖励 | 技能槽（装备使用）|

**注意**：技能系统已独立，不再作为卡牌类型。技能通过技能槽装备，使用回合冷却机制。

#### 草药卡（herb）

**基础属性**：
```javascript
{
  id: "herb_jingjie",
  name: "荆芥",
  type: "herb",              // herb / secret / magic
  category: "解表药",        // 药物分类
  rarity: "common",          // 品质：common/uncommon/rare/legendary
  qiCost: 1,                 // 精气消耗
  baseEffect: 8,             // 基础治疗效果
  effects: [
    { type: "heal", value: 8 }
  ],
  price: {                   // 价格（文）
    low: 5,
    middle: 10,
    high: 12
  },
  description: "祛风解表，透疹消疮",
  indication: "感冒，麻疹不透"
}
```

**品质影响**：
| 品质 | 名称 | 效果加成 | 外观 |
|------|------|---------|------|
| common | 常用药 | 基准 | 灰色边框 |
| uncommon | 较常用 | +10% | 蓝色边框 |
| rare | 名贵药 | +20% | 金色边框 |
| legendary | 稀世珍品 | +30% | 紫色边框 |

#### 秘方卡（secret）

**作用机制**：
- 收藏册中永久持有，不占用战斗牌库
- 战斗中自动检测组成条件
- 满足条件时触发组合技效果

**秘方结构**：
```javascript
{
  id: "formula_mahuang_tang",
  name: "麻黄汤",
  type: "secret",
  category: "经方",          // 经方/时方/验方
  rarity: "common",          // 稀有度
  requiredHerbs: ["mahuang", "guizhi", "xingren", "gancao"],
  comboEffect: {
    type: "heal_boost",
    multiplier: 2.0,         // 治疗效果×2
    extraEffect: "dispel_cold" // 额外驱散寒邪
  },
  description: "发汗解表，宣肺平喘",
  unlockVia: "cure_cold_wind" // 解锁条件：治愈风寒证
}
```

#### 巫术卡（magic）

**作用机制**：
- 祝由术法，非正统医术
- 特殊效果，可能带有副作用
- 通过特定NPC或黑市获取

**巫术结构**：
```javascript
{
  id: "magic_zhuyou",
  name: "祝由术",
  type: "magic",
  qiCost: 2,
  effects: [
    { type: "heal", value: 15 },
    { type: "random_side_effect", value: 1 }  // 随机副作用
  ],
  description: "以符咒治病，效果难料"
}
```

### 3.3 探索规则 — 五大区域系统

游戏世界分为五大区域，每个区域有独特的NPC、事件和资源。

#### 区域定义

| 区域 | 危险度 | 解锁条件 | 主要资源 | 特色NPC |
|------|--------|---------|---------|---------|
| **村庄** | ⭐ | 初始解锁 | 草药采集、基础任务 | 村长、农户、游方郎中 |
| **城镇** | ⭐⭐ | 初始解锁 | 商业交易、信息获取 | 药铺老板、商会、乞丐 |
| **官府** | ⭐⭐⭐ | 声望≥100 | 官府任务、高级药方 | 县官、捕快、师爷 |
| **山寨** | ⭐⭐⭐⭐ | 道德<0或完成特定任务 | 黑市、特殊草药 | 寨主、山贼、黑市商人 |
| **租界** | ⭐⭐⭐ | 完成主线"洋医挑战" | 西医卡牌、进口药 | 外国医生、传教士、买办 |
| **朝廷** | ⭐⭐⭐⭐⭐ | 声望≥500 | 御医任务、宫廷秘方 | 御医、太监、大臣 |

#### 探索机制

**基础规则**：
- 每次探索消耗1-3点精力值（根据区域危险度）
- 每日精力值上限10点，凌晨重置
- 探索结果随机生成，受声望、道德影响

**探索流程**：
```
选择区域 → 消耗精力 → 随机事件判定 → 结果结算
```

**事件类型分布**：
| 事件类型 | 概率 | 说明 |
|---------|------|------|
| 遭遇病人 | 25% | 可接取治疗任务 |
| 发现草药 | 20% | 采集草药资源 |
| 遇到NPC | 15% | 触发对话、任务 |
| 随机事件 | 15% | 奇遇、危机、抉择 |
| 战斗遭遇 | 10% | 与敌对势力战斗 |
| 无事件 | 15% | 安全返回 |

#### 区域详情

**村庄区域**：
- **特点**：民风淳朴，草药资源丰富
- **主要事件**：
  - 农户求助：治疗常见疾病
  - 草药采集：获得基础草药
  - 民间偏方：学习特殊药方
- **特殊地点**：
  - 药田：稳定获取草药
  - 祠堂：学习传统医术
  - 集市：低价交易

**城镇区域**：
- **特点**：商业繁华，信息集散地
- **主要事件**：
  - 药铺交易：购买/出售草药
  - 江湖消息：获取任务线索
  - 商会委托：大型治疗任务
- **特殊地点**：
  - 药铺：购买草药、药方
  - 茶馆：获取情报
  - 客栈：休息恢复

**官府区域**：

### 3.4 药品品质与价格系统

详细的药品经济系统，影响玩家收入和游戏策略。

#### 价格计算公式

**基础公式**：
```
草药价格 = 基础价格 × 品质倍率 × 市场波动

成药价格 = Σ(组成草药价格) × 2

治疗收费 = 主要药物成本 × 3 × 难度系数 × 议价调整
```

**品质倍率**：
| 品质 | 倍率 | 效果 | 获取难度 |
|------|------|------|---------|
| 下品 | ×0.5 | -20% | 容易 |
| 中品 | ×1.0 | 基准 | 普通 |
| 上品 | ×1.2 | +20% | 困难 |

**难度系数**：
| 病症难度 | 系数 | 说明 |
|---------|------|------|
| 简单 | ×1.0 | 普通感冒 |
| 一般 | ×1.2 | 常见疾病 |
| 困难 | ×1.5 | 疑难杂症 |
| 精英 | ×2.0 | 重症 |
| Boss | ×3.0 | 绝症 |

**议价调整**：
- 基础价格可议价 ±20%
- 声望越高，议价能力越强
- 道德值影响：正道NPC对低道德玩家提价

#### 经济循环

**收入来源**：
| 来源 | 比例 | 说明 |
|------|------|------|
| 治疗收费 | 60% | 主要收入来源 |
| 草药出售 | 20% | 采集多余草药 |
| 任务奖励 | 15% | 完成各类任务 |
| 其他 | 5% | 奇遇、投资等 |

**支出项目**：
| 项目 | 比例 | 说明 |
|------|------|------|
| 健康维持 | 30% | 每日生活成本 |
| 草药采购 | 40% | 补充治疗用药 |
| 装备购买 | 15% | 技能、道具 |
| 其他 | 15% | 学习、社交等 |

**经济平衡设计**：
- 早期：入不敷出，需要精打细算
- 中期：收支平衡，开始积累
- 后期：收入丰厚，财务自由

## 四、战斗核心机制

### 4.1 战斗框架

```

  [病症区] ←—— 每回合释放攻击 ——→ [病人（玩家）]
                                         ↑
```

**病人健康值**：0 ~ 200（初始 100，达到 200 = 治愈胜利，降到 0 = 死亡失败）

## 五、战斗数值体系

### 设计锚点与校验

| 锚点 | 数值 | 校验逻辑 |
|------|------|---------|
| H_ref（治疗基准） | **25 HP/回合** | 3气×方剂草药均值9.2=27.6，保守取25 |
| 病人健康值上限 | **200 HP** | ≈ 8×H_ref，8回合充分治疗可治愈 |
| 病人净增需求 | **100 HP**（100→200）| ≈ 4回合H_ref，方剂加速可1-2回合完成 |
| 方剂最优单回合 | **45 HP**（君+臣+佐，3气）| 加组合技bonus → 65-75 HP |
| 方剂大招参考值 | **50 HP**（君+臣+佐+使全和）| 与病症大招伤害对应 |

```
校验1：普通草药5-8个回合攻击值和 ≈ 病人健康值
  5 × 25 = 125 ≈ 100（净增）
  8 × 25 = 200 = 上限 ✓

校验2：常用方剂1-2个回合攻击值和 ≈ 病人净增需求
  1.5回合 × 65（方剂+组合技）= 97 ≈ 100 ✓
```

### 疾病普通攻击设计

**公式：每回合攻击 = D_base × rand(0.8, 1.2)**

| 难度 | D_base/回合 | 随机范围 | H_ref比值 | 中等玩家净值/回合 |
|------|------------|---------|----------|----------------|
| 一般 | 20 | 16–24 | ×0.80 | **+8** (教学关卡，~75%胜率) |
| 较难 | 25 | 20–30 | ×1.00 | **+3** (目标难度，~55%胜率) |
| 极难 | 31 | 25–37 | ×1.24 | **−3** (需方剂/技能，~30%胜率) |
| 绝症 | 38 | 30–46 | ×1.52 | **−10** (需精准操作+运气，<15%) |

> 中等玩家 H_actual ≈ 28（合理出牌，偏向方剂草药）；基准玩家 H = H_ref = 25

**"略大于50%"的来源（较难·中等玩家）：**
- 主动选方剂草药打出：H_actual = 28 > D_mean = 25
- 净 +3/回合 提供略微正期望
- 方剂组合技偶发（每场1-2次）提供额外缓冲
- 运气方差（D波动±20%）决定最终结果

### 疾病大招系统（Disease Ultimate Attack）

**设计原则：病症的大招 = 对应方剂的"邪气反攻"**

每种病症有且仅有一个大招，其伤害值以该病症的代表方剂全草药之和（君+臣+佐+使）为基础，乘以难度系数。

**大招伤害公式：**
```
大招伤害 = formulaHerbSum × difficultyMultiplier

formulaHerbSum = 对应方剂的（君×4 + 臣×3 + 佐×2 + 使×1）总和
  示例：麻黄汤 → 20+15+10+5 = 50
  示例：六味地黄丸（6味）→ 依方剂实际计算

难度系数：
  一般  → ×0.9   （示例麻黄汤：50×0.9 = 45）
  较难  → ×1.2   （50×1.2 = 60）
  极难  → ×1.8   （50×1.8 = 90）
  绝症  → ×2.5   （50×2.5 = 125，可能一击致死！）
```

**大招触发参数：**

| 难度 | 冷却（回合） | 预告回合数 | 设计意图 |
|------|-----------|---------|---------|
| 一般 | 8 | 1 | 教学提示，可轻松应对 |
| 较难 | 5 | 1 | 需留意意图；方剂回血可解 |
| 极难 | 4 | 2 | 2回合准备期；没有技能极危险 |
| 绝症 | 3 | 2 | 近乎无解，必须有技能槽兜底 |

**大招 ↔ 方剂的镜像关系（设计美感）：**
```
方剂治疗力 ←→ 病症大招伤害

   玩家用麻黄汤治病 → 回血50 HP
   病症大招（较难）  → 伤害60 HP

   即：正确方剂能抵消约85%的大招伤害
       技能（如符咒·镇煞）用于拦截极难/绝症大招
```

**绝症大招特别说明（~125 HP 一击）：**
- 病人从100 HP起始，普通进攻后HP约在 100-130 范围
- 绝症大招 125 HP 可瞬杀低HP情况 → 必须配合：
  - `符咒·镇煞`（免疫一回合，冷却5回合）
  - `叫魂`（legendary，死亡后以1血复活）
  - 提前积累HP缓冲（先方剂冲到150+再迎大招）

### 方解解析规则

```
原始: "君：麻黄；臣：桂枝；佐：杏仁；使：炙甘草"
规则: 以"；"分割角色组，每组格式"角色：药A、药B"
角色: 君 / 臣 / 佐 / 使（4个，部分方剂可能缺某角色）
```

### 草药牌攻击值

**基础值**: 每张草药牌 = 5

```
君（×4）= 20    ——  决定治疗方向
臣（×3）= 15    ——  加强主药
佐（×2）= 10    ——  调和配伍
使（×1）= 5     ——  引经，基础
```

**随机草药牌**（非对症方剂草药，占战斗牌池约 30%）：
- 定义：不在当前战斗对应方剂君臣佐使列表中的草药
- 攻击值：随机整数 **-5 ~ +8**（负值 = 配伍禁忌，误用反伤）

**草药名称标准化**：
- 精确匹配（formula 简体 → herbs 简体/繁体混存）
- 二级匹配：去修饰词前缀（炙、生、制、去皮等）
- 失败降级：使用占位图

**运行时反向索引**（BootScene 构建）：
```javascript
// herbRoleMap[formulaName][herbName] = '君'|'臣'|'佐'|'使'
// BattleScene 出牌时查表确定攻击乘数
this.registry.set('herbRoleMap', buildHerbRoleMap(formulas))
```

### 药剂系统（v1.1，MVP 不做）

- **前提**：玩家持有对应方剂（战斗/探索获取）
- **攻击力**：`(构成草药攻击力之和) × 配伍乘数`
  - 4种角色全齐 → ×10；缺1种 → ×5；缺2种 → ×2
- **示例**：麻黄汤 (20+15+10+5=50) × 10 = 500攻击

```
                               runDuration: number }
ResultsScene→ MenuScene     无数据（run 状态清空）
```

```
Key: "medgod_run_state"

清空: ResultsScene 点击"重试"或"返回菜单"时
```

## 十、技能槽系统

### 架构定义

技能槽**独立于草药牌组**，使用**回合冷却**激活（不消耗精气）。采用**按钮+展开面板**的交互设计，避免遮挡手牌区。

```
战斗界面布局:
  [手牌区：草药牌 × 5]
  
  左下角:
    [技能 ▲]  ← 点击展开/收起（快捷键 S）
      ↓ 展开后面板
    ┌─────────────┐
    │   选择技能   │
    │   [Q] [W]   │
    │  [技能槽1]   │  ← 点击使用（快捷键 Q）
    │  [技能槽2]   │  ← 点击使用（快捷键 W）
    └─────────────┘
```

### 系统参数

| 参数 | 值 |
|------|----|
| 槽位数（MVP） | 2 |
| 激活代价 | 回合冷却，不消耗精气 |
| 冷却计时规则 | 使用当回合立即进入冷却；**回合结束时 cd-1**；cd 归零后下回合开始可用 |
| Legendary 技能 | cooldown = 999（每场战斗仅能使用 1 次） |
| 跨战斗冷却 | 每场战斗开始时所有 cd 重置为 0 |
| 单回合效果持续 | 仅持续当回合（如"本回合精气+1"），回合结束自动失效 |
| UI 形式 | 按钮+展开面板，默认收起，点击后展开 |
| 按钮位置 | 左下角 (60, 500)，绿色背景 |
| 面板位置 | 按钮上方，垂直排列 2 个技能槽 |
| 快捷键 | S: 展开/收起面板；Q: 使用槽1；W: 使用槽2；ESC: 关闭面板 |
| 使用反馈 | 使用技能后面板自动关闭，显示技能名称动画 |

### 反 AI slop 检查清单

❌ 紫色渐变背景 ❌ 圆形图标+3列布局 ❌ 居中对齐所有元素

### 响应式与无障碍（Pass 6）

**响应式（MVP 范围）**：
- 桌面优先，960×640px 固定 Phaser canvas
- CSS scale transform 保证视口 < 960px 时等比缩放：`transform: scale(min(1, viewportW/960))`
- 不为移动端单独设计交互（MVP 不做触摸优化，见 Open Questions 决策记录）

**键盘快捷键（已在 BattleScene.js 实现）**：
```
数字键 1-5：选中第 N 张手牌（再按一次=打出）
ENTER：打出已选牌
E：结束回合
ESC：取消选中
```

**颜色无障碍**：
- 牌类型区分不单靠颜色（有文字标签兜底）
- HP 进度条：颜色 + 数字双渠道
- 胜败：颜色 + 大字标题 + 结构布局区分
- 最低对比度目标：正文文字 4.5:1（`#4a4a4a` on `#f5f1e8` ≈ 7:1 ✓）
