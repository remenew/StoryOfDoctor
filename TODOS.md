### 4. Add performance profiling hooks to critical paths
**What**: 在关键路径添加console.time()标记，用于性能调试

**Why**: 如果游戏出现卡顿，可以快速识别瓶颈所在

**Pros**:
- 快速添加（约10分钟）
- 生产环境零成本（可用Vite构建工具剥离）
- 帮助早期诊断性能问题

**Cons**:
- 最小开销 - 需要记得移除或配置构建剥离

**Implementation**:
```javascript
// BattleScene.js
update(time, delta) {
  console.time('BattleScene.update');
  // ... game logic
  console.timeEnd('BattleScene.update');
}

// DeckSystem.js
drawCards(count) {
  console.time('DeckSystem.drawCards');
  // ... draw logic
  console.timeEnd('DeckSystem.drawCards');
}
```

**Critical paths to profile**:
- BattleScene.update() (每帧调用)
- DeckSystem.drawCards() (回合开始)
- ComboSystem.checkCombos() (每次出牌)
- Patient.modifyHealth() (健康值变化)

**Context**: Phaser游戏可能有微妙的性能问题（过多精灵、未优化的渲染、频繁的垃圾回收）
